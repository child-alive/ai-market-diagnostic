## 2. 改了什么

### 性能与稳定性

- 删除 Google Fonts 和 jsDelivr ECharts 两个阻塞型外部依赖；
- 完整报告的固定 SOV 图改为轻量 CSS 图表，数据与无障碍描述保持一致；
- 技术视角、实况、原始 JSON 查看器改为异步组件；
- 首屏增加 CSS 骨架，并预加载 221KB 回放 JSON；
- HTML/CSS/JS/JSON 构建后生成 `.gz`，Nginx 启用 `gzip_static`；
- 维持系统字体栈，不增加 UI 库、图表库或运行时依赖。

### 评审体验

- Hero 用一句话解释产品，再给出“AI 认识 Deli，但不会主动推荐它”的核心洞察；
- 首屏展示无品牌词提及 0%、品牌词可见度 100%、官网西语内容 0 页和单轮样本边界；
- 以测试题原文六项考察为产品视角的主目录，每项在原地展开“回答 + 可核验证据”；
- 删除独立的“范围与取舍”等内部自述板块，只在测试题明确询问处给出必要证据；
- 产品视角只讲业务问题、核心洞察、诊断过程和客户行动；
- 技术视角作为独立模块，只讲架构、数据契约、可靠性、部署边界和可复现命令；
- 国内 / 国际不再伪装成网络自动识别：同一页面、两个显式 URL、仅 API 能力不同。
- 页面中彻底移除线路、部署、URL 回填与迭代说明；这些内容只留在提交和部署文档，不占用
  HR / 产品 / 技术评审的阅读空间。
- 产品视角把 Fanout、Mock、Pydantic 和错误代码翻译为派生问法、离线复现、统一数据契约和
  面向业务的行动标题；技术视角保留精确术语。

### 交互与适配

- 吸顶导航切换真正独立的 `ProductView` / `TechnicalView`，并用 `?view=product|technical` 分享指定视角；
- 回放支持暂停、1×/2×、逐步和跳到结论；
- Hero“从头播放”会重置到 1/7、恢复播放并滚到流程区；
- 进度线按首尾节点中心计算，桌面实测两端偏差 0px，节点有 hover / active 反馈；
- 报告入口使用当前页面同源绝对地址，生产预览实测命中 `/report/`，不再落到旧 5173 页面；
- 全局加入首屏分层进入、视角切换、滚动渐显、卡片悬停和节点状态过渡；支持
  `prefers-reduced-motion` 降级。
- 六项考察改为单项展开组件，使用 CSS Grid 自动高度过渡；1920px 下内容容器宽 1200px，
  左右留白各 360px、中心偏差 0。
- 产品视角新增面向非技术评审的“现场跑一次”；与技术视角共享同一受控 SSE 实况能力，
  但使用业务语言解释问题、结果和额度边界。
- 390×844 手机竖屏无横向溢出；
- 新增 favicon、强化 title/meta、自包含 404；
- `/report/` 在国内与国际两套产物中均为直接可达的完整报告。

## 4. 两条部署线路

| 线路 | 产物 | 能力 | 部署 |
|---|---|---|---|
| 国内动态版 | `web/dist/` | 回放 + 完整报告 + 显式实况 `/api` | `deploy/部署手册.md` + ECS/Nginx |

Cloudflare Direct Upload 的当前控制台流程与限制以
[官方文档](https://developers.cloudflare.com/pages/get-started/direct-upload/)为准。
