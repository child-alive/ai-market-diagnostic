# 双受众演示网页 · ECS 部署手册

适用环境：Ubuntu / Debian、2 核 2G、可 SSH 的公网 IP。部署结果通过
`http://<公网 IP>:<端口>/` 访问，不要求域名或备案。默认端口是 `8080`。

## 1. 上线前边界

- `web/dist/` 是已构建的静态回放页，**不需要 API key，始终可用**；
- FastAPI 只负责实况模式：固定 3~5 个仓库内无品牌词，不接受用户任意 Prompt；
- DeepSeek key 只放服务器 `.env`，不会进入 Vue 源码、构建产物、浏览器响应或部署脚本；
- 应用层限制每 IP 每小时 2 次、全局并发 1；Systemd 必须保持 `--workers 1`，否则
  每个 worker 会有独立的内存限流器；
- 总超时到达后 API 会终止整个子进程，避免请求在后台继续消耗额度；
- 主 CLI、SQLite 与提交报告不依赖网页，网页异常不会破坏主交付物。

## 2. 本机准备

在项目根目录确认构建和测试：

```bash
cd web
npm ci
npm run build:domestic
cd ..
.venv/bin/python -m pytest -q
```

部署脚本不会上传本机 `.env`，也不会上传 SQLite、临时 `data/report.*`、`.venv` 或
`node_modules`。`data/示例产物` 与 `web/dist` 会上传。

## 3. 一键部署

确保云安全组已放行 TCP `8080`（或你选择的端口），然后运行：

```bash
./deploy/deploy.sh ubuntu@<公网 IP> 8080
```

脚本会在远端完成：

1. 安装 Python venv 与 Nginx；
2. 安装 `requirements.txt`；
3. 创建单 worker Systemd 服务 `ai-market-diagnostic`；
4. 让 Nginx 托管 `web/dist`，把 `/api/` 反向代理到 `127.0.0.1:8000`；
5. 关闭 SSE 代理缓冲、设置 190 秒读取超时；
6. 若 UFW 已启用，只追加 SSH 与所选网页端口规则；脚本不会主动启用 UFW。

首次部署若服务器没有 `.env`，网页回放照常可用，实况按钮会收到 503 并提示回放。

## 4. 只在服务器配置 DeepSeek

SSH 登录服务器：

```bash
ssh ubuntu@<公网 IP>
sudo install -o root -g root -m 600 /dev/null /opt/ai-market-diagnostic/.env
sudo nano /opt/ai-market-diagnostic/.env
```

填写：

```dotenv
DEEPSEEK_API_KEY=你的真实_key
DEEPSEEK_BASE_URL=https://api.deepseek.com
DEEPSEEK_MODEL=deepseek-v4-flash
DEEPSEEK_WEB_SEARCH=true
DEEPSEEK_SEARCH_MAX_USES=3
DEMO_RATE_LIMIT_PER_HOUR=2
DEMO_LIVE_TIMEOUT_SECONDS=180
DEMO_TRUST_PROXY=true
```

保存后：

```bash
sudo systemctl restart ai-market-diagnostic
sudo systemctl status ai-market-diagnostic --no-pager
```

不要把服务器 `.env` 下载回仓库，也不要把 key 放进 Nginx、Vue 或 URL 参数。

## 5. 验收

```bash
curl -sS http://<公网 IP>:8080/api/health
```

预期：

```json
{"status":"ok","replay_available":true,"live_available":true,"live_question_range":[3,5]}
```

浏览器检查：

1. 默认回放自动推进七个诊断阶段；
2. 产品视角 / 技术视角可切换；
3. `/report` 与页面内“完整报告”能打开固化的 `37b442ec`；
4. 实况模式点击前不产生请求；点击后逐题流式出现，最终显示 `DONE`；
5. 连续第三次运行返回 429，已有任务运行时第二个请求返回 409；
6. 暂时移除 key 后重启服务，回放仍可用且实况返回友好 503。

## 6. 运维与排错

```bash
# 服务日志（代码不会记录 key）
sudo journalctl -u ai-market-diagnostic -n 100 --no-pager

# Nginx 检查
sudo nginx -t
sudo tail -n 100 /var/log/nginx/error.log

# 重启
sudo systemctl restart ai-market-diagnostic nginx
```

- `live_available=false`：检查服务器 `.env` 是否存在且为 `root:root 600`（systemd 以 root 读取 EnvironmentFile，服务进程无需读权限），随后重启服务；
- 实况 409：另一个任务仍在运行；等待或使用回放；
- 实况 429：该 IP 一小时内已运行两次；这是预期防护；
- 实况超时：上游搜索慢或不可用，子进程会被终止，回放不受影响；
- 页面字体使用系统字体栈，不依赖 Google Fonts 或外部 CDN；
- 重复部署：重新运行 `deploy.sh` 即可；它不会覆盖远端 `.env`。

## 7. 安全说明

- 云安全组只开放 SSH 和网页端口；FastAPI 的 `8000` 只监听 loopback，不对公网开放；
- 无域名时使用 IP:端口是明文 HTTP，但 key 从不传到浏览器；如后续绑定域名，应加 HTTPS；
- 当前限流是单机/单 worker内存窗口，适合 2 核 2G 评审演示。正式多实例部署应改 Redis
  等共享限流并增加身份认证；
- 前端展示的是公开搜索回答与公开来源链接，不应加入内部客户数据；
- 如需立即关闭实况但保留回放：从服务器 `.env` 移除 `DEEPSEEK_API_KEY` 并重启服务。
