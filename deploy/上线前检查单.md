# 上线前检查单（阿里云 ECS · IP:端口 · 无域名）

> 配套：`deploy/部署手册.md`（完整步骤）、`deploy/deploy.sh`（本机执行）、`deploy/install_remote.sh`（远端执行）。
> 以下以 `ECS_IP` 代指公网 IP，`PORT` 代指对外端口（默认 8080）。

## 一、部署前（本机 + 阿里云控制台）

- [ ] **阿里云安全组**：入方向放行 `PORT/tcp`（授权对象 0.0.0.0/0）与 `22/tcp`（建议限定为你自己的出口 IP）；**不要**放行 8000（FastAPI 只监听 127.0.0.1，无需对外）
- [ ] **本机检查**：`bash -n deploy/deploy.sh deploy/install_remote.sh` 语法通过；`web/dist/index.html`、`web/dist/report/index.html`、`web/dist/demo-report.json` 存在且为最新构建
- [ ] **确认 rsync 排除**：`deploy.sh` 已排除 `.env`/`.git`/`.venv`/`node_modules`/`diagnostic.db`——部署不会上传或覆盖服务器上的任何密钥

## 二、部署执行

- [ ] 本机运行 `bash deploy/deploy.sh root@ECS_IP [PORT]`（内部完成 rsync + 远端 `install_remote.sh`）
- [ ] **手工配置服务端密钥**（脚本刻意不做这步）：SSH 登录后创建 `/opt/ai-market-diagnostic/.env`，只需 `DEEPSEEK_API_KEY=...`（及可选的 `DEEPSEEK_WEB_SEARCH=true`）；然后执行：
  ```bash
  chown root:root /opt/ai-market-diagnostic/.env && chmod 600 /opt/ai-market-diagnostic/.env
  systemctl restart ai-market-diagnostic
  ```
  说明：systemd 以 root 读取 EnvironmentFile 后再降权到 www-data 运行，600 权限不影响服务读取。不配置密钥时网页回放与报告仍完整可用，仅实况模式返回友好提示（fail-closed）。

## 三、服务器侧验证（SSH 内）

- [ ] `systemctl is-enabled ai-market-diagnostic nginx` → 两项均为 `enabled`（开机自启）；`systemctl status ai-market-diagnostic nginx` → 两项均为 `active (running)`
- [ ] `systemctl show ai-market-diagnostic -p Restart` → `Restart=always`（异常退出与正常退出均自动拉起）
- [ ] `nginx -t` 通过；`ss -tlnp | grep -E '8000|PORT'` → uvicorn 只在 `127.0.0.1:8000`，nginx 在 `0.0.0.0:PORT`
- [ ] `curl -fsS http://127.0.0.1:8000/api/health` → 返回 JSON，`replay` 可用；配置密钥后 `live` 也为可用
- [ ] 若启用了 ufw：`ufw status` 只有 OpenSSH 与 `PORT/tcp`

## 四、外网自测（你自己的电脑）

- [ ] `curl -I http://ECS_IP:PORT/` → 200（Vue 回放页）
- [ ] `curl -I http://ECS_IP:PORT/report` → 302 到 `/report/`；`curl -I http://ECS_IP:PORT/report/` → 200（静态报告直达）
- [ ] `curl -fsS http://ECS_IP:PORT/api/health` → 200 JSON（反代链路通）
- [ ] `curl -I http://ECS_IP:PORT/.env` → 403/404（隐藏文件被拒，顺手验证）
- [ ] 浏览器打开 `http://ECS_IP:PORT/`：回放自动播放、"打开完整报告"可跳转、控制台无报错
- [ ] **限流生效验证**（会消耗 2 次真实实况额度，可选）：网页上连续启动实况 2 次成功后，第 3 次应收到"每 IP 每小时 2 次"的拒绝提示；或命令行连续三次
  `curl -N -X POST http://ECS_IP:PORT/api/live-diagnose/stream -H 'Content-Type: application/json' -d '{"question_limit":3}'`，第 3 次应返回 429
- [ ] 手机流量（非同一 Wi-Fi）访问一次，确认公网可达

## 五、收尾

- [ ] 把 `http://ECS_IP:PORT/` 回填到 `SUBMISSION.md` 在线演示行，并删除"未部署则删除本行"提醒
- [ ] `git add SUBMISSION.md && git commit`，重跑 57 个测试与 `--mock` 基线做最终冻结

## 常见问题

| 症状 | 排查 |
| --- | --- |
| 外网打不开但 curl 本机 200 | 阿里云安全组未放行 PORT（最常见），其次 ufw |
| /api/health 200 但实况 503 | `.env` 未配置或服务未 restart；`journalctl -u ai-market-diagnostic -n 50` |
| 实况点击后立即失败 | 服务器出网访问 DeepSeek API 受限，看 journalctl 中 httpx 错误 |
| 报告页样式异常 | 确认 `web/dist/report/index.html` 为最新构建；报告无 CDN 依赖，不需要访问外网资源 |
